開始冒險之旅 學習 AI 魔法

GitHub 帳號申請與使用

這是開發者的履歷與基地，來蓋你的第一棟房子吧！

1 新手村註冊

訪問官網

前往 github.com，點擊右上角那個顯眼的 "Sign up" 按鈕。

選擇註冊方式

最快的方法：點選 Google 按鈕。
或是手動填寫 Email 來建立你的冒險者 ID。

驗證與方案

證明你不是機器人（雖然我們喜歡機器人）。選擇 "Free" 免費方案就非常夠用囉！

2 建立秘密基地 (Repo)

新增專案

點擊右上角 "+" 號，選擇 New repository。

設定內容

  • Name: 幫專案取個名字 (例: my-site)
  • Public: 公開讓大家看見你的作品
  • 勾選 Add a README file (很重要!)

上傳檔案

點擊 Add file > Upload files。把你的網頁檔案拖進去，按下綠色的 "Commit changes" 就成功啦！

AI 魔法教室

利用 Gemini 設計教學網頁

學會「咒語 (Prompt)」，讓 AI 成為你的專屬程式設計師。

召喚強大 AI 顧問前的準備

把這兩張「魔法卡牌」丟給 Gemini，它就會進化成超級專家模式喔！

卡牌 1：事實查核盾

在回答接下來的問題之前，請先執行「內部事實查核」程序： 【證據優先】：僅依據資料庫中『確切已知』的事實回答，嚴禁使用『可能、應該、或許』等模糊推測。 【允許留白】：如果信心水準低於 90%，或資料庫中缺乏相關資訊，請直接輸出：「【資料不足，無法確認】」。 【來源標註】：如果涉及具體數據或法規，請務必列出參考來源或推論邏輯。

卡牌 2：優化詠唱流

我的回應流程包含兩個步驟： 第一步，我會作為AI領域的顧問，根據你提供的原始提示詞進行優化，使其轉化為更精確、高效能的指令。優化時會遵循以下原則： * 最有效的提示詞平均約為21個字。 * 若原始提示詞過於簡短，將強化「背景資訊」維度，以確保回覆品質。 * 轉化時必須包含以下四個維度：[角色任務]、[背景資訊]、[具體指令]、[約束條件]。 第二步，優化完成後，我會立即依照優化後的指令進行答覆，產出最終內容。

1. 設定人設

告訴 Gemini：「你現在是全世界最厲害的網頁設計大師！」

2. 提出需求

想要什麼顏色？什麼功能？像是點餐一樣說清楚。

3. 修改調整

不滿意？直接說：「這裡太擠了、那裡字太小」，讓它改到好。

🔥 萬用咒語範本

複製這段文字，填入你的想法，見證奇蹟的時刻。

# 角色 你是一位資深的網頁前端設計師。 # 任務 請幫我用 HTML5 與 Tailwind CSS (CDN版本) 製作一個單頁式網頁。 # 內容需求 1. 主題：[填入你的主題，例如：貓咪咖啡廳] 2. 區塊：導覽列、Hero區塊(大圖與標題)、特色介紹(3欄式卡片)、頁尾。 3. 風格：[填入風格，例如：可愛、粉色系]。 # 限制 所有 CSS 都要寫在 class 裡，確保 RWD 手機版也能正常顯示。
